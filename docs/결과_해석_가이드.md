# 중복 탐지 결과 해석 가이드

## 📊 결과 파일 구조

### 주요 컬럼 설명

#### 1. **vendor_id** (맨 앞 컬럼)
- 원본 데이터의 행 인덱스
- 각 행을 고유하게 식별하는 ID

#### 2. **old_group**
- 이전 그룹번호 (현재는 None)
- 향후 업데이트 시 이전 그룹 정보를 추적하기 위한 컬럼

#### 3. **그룹번호_STEP1**
- STEP1 후보 그룹 번호
- **의미**: 중복 가능성이 있어 추가 검토가 필요한 후보 그룹
- 같은 번호를 가진 행들은 중복일 가능성이 있지만, 아직 확정되지 않음
- STEP2에서 최종 판단되지 않은 후보 그룹도 포함

#### 4. **그룹번호_STEP2**
- STEP2 최종 중복 그룹 번호
- **의미**: 실제로 같은 업체로 판단된 그룹
- 같은 번호를 가진 행들은 **같은 업체**로 확정됨
- 중복이 아닌 행은 `None` 또는 빈 값

#### 5. **match_type** ⭐ 중요
매칭 타입을 나타내는 코드:

- **`NORMALIZED_NAME_MATCH`**
  - 정규화된 이름이 동일한 경우
  - 예: "LEEPACK CO., LTD" vs "LEEPACK CO.,LTD"
  - 구두점/공백/법인형태 차이만 있는 케이스

- **`SAME_COMPANY_DIFFERENT_LOCATION`**
  - 동일 법인이지만 다른 사업장
  - 정규화된 이름 동일 + 국가 동일 + 주소 다름
  - 예: "LEEPACK" (인천) vs "LEEPACK" (서울)

- **`CORE_NAME_MATCH`**
  - 고유 상호가 일치하는 경우
  - 가장 일반적인 매칭 타입
  - 고유 상호 추출 후 비교하여 일치

- **`CORE_NAME_MISMATCH`**
  - 고유 상호가 불일치 (중복 아님)

- **`LEGAL_FORM_CONFLICT`**
  - 법인 형태 충돌 (중복 아님)
  - 예: "建设工程总公司" vs "股份有限公司"

- **`BUSINESS_TYPE_CONFLICT`**
  - 업종 충돌 (중복 아님)

- **`INSUFFICIENT_EVIDENCE`**
  - 증거 부족 (중복 아님)

#### 6. **match_reason** ⭐ 중요
매칭 이유를 한국어로 상세히 설명:

예시:
- `"고유 상호 일치: 0.875, 주소 유사도: 0.449, 이름 유사도 매우 높음"`
- `"정규화된 이름 동일: LEEPACK, 국가 동일: KR, 주소 다름 (다른 사업장), 주소 유사도: 0.25"`

**해석 방법**:
- 숫자가 높을수록 유사도가 높음 (0.0 ~ 1.0)
- 여러 조건이 조합되어 최종 판단됨

#### 7. **confidence** (신뢰도)
- 중복 판단의 신뢰도 (0.0 ~ 1.0)
- **0.9 이상**: 매우 높은 신뢰도 (거의 확실)
- **0.7 ~ 0.9**: 높은 신뢰도
- **0.5 ~ 0.7**: 중간 신뢰도 (검토 권장)
- **0.5 미만**: 낮은 신뢰도 (재확인 필요)

#### 8. **_duplicate_count**
- STEP2 최종 그룹의 행 수
- 해당 그룹에 몇 개의 중복 행이 있는지 표시
- 중복이 아닌 행은 `None`

## 🔍 결과 확인 방법

### Excel에서 확인하기

#### 1. 최종 중복 그룹 확인
1. `그룹번호_STEP2` 컬럼으로 정렬 (내림차순)
2. 값이 있는 행들이 최종 중복 그룹
3. 같은 번호를 가진 행들이 같은 업체

#### 2. 후보 그룹 확인
1. `그룹번호_STEP1` 컬럼으로 필터링
2. STEP2에서 제외된 후보 그룹도 확인 가능

#### 3. 매칭 타입별 확인
1. `match_type` 컬럼으로 필터링
2. 특정 타입의 매칭만 확인 가능
   - 예: `SAME_COMPANY_DIFFERENT_LOCATION`만 필터링하여 다른 사업장 케이스 확인

#### 4. 신뢰도별 확인
1. `confidence` 컬럼으로 정렬 (내림차순)
2. 높은 신뢰도부터 확인하여 우선 검토

## 📋 실제 해석 예시

### 예시 1: 정규화된 이름 일치
```
그룹번호_STEP2: 5
match_type: NORMALIZED_NAME_MATCH
match_reason: 정규화된 이름 동일: LEEPACK, 국가 동일: KR, 주소 유사도: 0.75
confidence: 0.98
```
**해석**: 
- "LEEPACK CO., LTD"와 "LEEPACK CO.,LTD"는 구두점 차이만 있음
- 정규화 후 동일한 이름으로 판단됨
- 주소도 유사하여 같은 사업장으로 판단
- 신뢰도 매우 높음 (0.98)

### 예시 2: 동일 법인, 다른 사업장
```
그룹번호_STEP2: 10
match_type: SAME_COMPANY_DIFFERENT_LOCATION
match_reason: 정규화된 이름 동일: LEEPACK, 국가 동일: KR, 주소 다름 (다른 사업장), 주소 유사도: 0.25
confidence: 0.92
```
**해석**:
- 같은 회사이지만 다른 지역에 있는 사업장
- 정규화된 이름이 동일하고 국가가 같음
- 주소가 다르므로 다른 사업장으로 판단
- 신뢰도 높음 (0.92)

### 예시 3: 고유 상호 일치
```
그룹번호_STEP2: 124
match_type: CORE_NAME_MATCH
match_reason: 고유 상호 일치: 0.875, 주소 유사도: 0.449, 이름 유사도 매우 높음
confidence: 0.58
```
**해석**:
- 고유 상호가 87.5% 일치
- 주소 유사도는 44.9% (중간 수준)
- 이름 유사도가 매우 높음
- 신뢰도 중간 (0.58) - 검토 권장

## ⚠️ 주의사항

### 1. 신뢰도가 낮은 경우 (0.5 미만)
- 재확인이 필요할 수 있음
- `match_reason`을 자세히 확인하여 판단 근거 검토

### 2. match_type이 없는 경우
- STEP2 최종 그룹이 아닌 경우
- 후보 그룹이거나 중복이 아닌 경우

### 3. 그룹번호_STEP1만 있고 STEP2가 없는 경우
- 후보 그룹이었지만 최종 판단에서 제외됨
- 실제로는 중복이 아닐 가능성이 높음

### 4. 여러 행이 같은 그룹번호를 가진 경우
- 모두 같은 업체로 판단됨
- 하나의 대표 업체로 통합하는 것을 고려

## 💡 활용 팁

### 1. 우선순위 확인
1. `confidence` 높은 순으로 정렬
2. 높은 신뢰도부터 검토

### 2. 특정 케이스 찾기
- 다른 사업장: `match_type = SAME_COMPANY_DIFFERENT_LOCATION`
- 구두점 차이: `match_type = NORMALIZED_NAME_MATCH`
- 고유 상호 일치: `match_type = CORE_NAME_MATCH`

### 3. 그룹별 확인
- `그룹번호_STEP2`로 그룹화하여 같은 그룹의 모든 행 확인
- 각 그룹의 `match_reason`을 비교하여 판단 근거 확인

### 4. 재확인 필요한 케이스
- `confidence < 0.5`
- `match_reason`에 "충돌" 또는 "불일치"가 포함된 경우
- 주소 유사도가 매우 낮은 경우 (0.3 미만)
