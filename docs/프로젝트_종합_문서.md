## 프로젝트 개요

### 목적
다국어 협력사 데이터(44,313행)에서 중복 업체를 탐지하고 그룹번호를 부여하는 시스템

### 핵심 목표
- ✅ **다국어 중복 탐지**: 같은 업체의 다른 언어 표기도 탐지
- ✅ **고성능 처리**: 44k 행을 실용적인 시간 내 처리
- ✅ **높은 정확도**: 오매칭 최소화 (Precision 우선)
- ✅ **확장 가능성**: 다양한 언어 및 데이터 특성에 대응

### 데이터 특성
- **데이터 크기**: 약 44,313행, 22열
- **지원 언어**: 중국어, 영어, 한글, 일본어, 포르투갈어, 베트남어, 인도네시아어, 태국어 등
- **핵심 문제**: 같은 업체가 다른 언어로 표기된 경우도 중복으로 탐지해야 함

## 시스템 아키텍처

### 전체 구조

```
┌─────────────────────────────────────────────────────────────┐
│                    입력: DataFrame (44,313행)                │
└──────────────────────┬──────────────────────────────────────┘
                       │
                       ▼
        ┌──────────────────────────────────────┐
        │  1. 전처리 및 메타데이터 생성           │
        │  - row_cache: 전처리 결과 캐싱         │
        │  - row_meta: ANN용 메타데이터          │
        │  - 임베딩 사전 계산 및 캐싱             │
        └──────────────────┬───────────────────┘
                           │
                           ▼
        ┌──────────────────────────────────────┐
        │  2. 후보 생성 (2가지 모드)            │
        │  [Legacy 모드] / [ANN 모드]            │
        └──────────────────┬───────────────────┘
                           │
                           ▼
        ┌──────────────────────────────────────┐
        │  3. 2단계 필터링 파이프라인           │
        │  STEP 1: Cheap Gate / STEP 2: Heavy Judge │
        └──────────────────┬───────────────────┘
                           │
                           ▼
        ┌──────────────────────────────────────┐
        │  4. 그룹화 (Union-Find)              │
        └──────────────────┬───────────────────┘
                           │
                           ▼
        ┌──────────────────────────────────────┐
        │  5. 결과 생성 → output/ 저장          │
        └──────────────────────────────────────┘
```
