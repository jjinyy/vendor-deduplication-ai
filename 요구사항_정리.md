# 협력사 중복 탐지 프로젝트 요구사항 정리

## 핵심 요구사항

### 1. 데이터 특성
- **데이터 크기**: 약 44,313행
- **데이터 형식**: CSV/Excel
- **컬럼**: 구매조, 공급업체코드, Land(국가코드), 공급업체명, 생성일, 주소 필드들 등

### 2. 다국어 지원 요구사항 ⭐ 핵심
- **지원 언어**: 중국어, 영어, 한글, 베트남어, 포르투갈어, 일본어 등
- **핵심 문제**: **같은 업체가 다른 언어로 표기된 경우도 중복으로 탐지해야 함**
  - 예시:
    - "阿里巴巴集团" (중국어) ↔ "Alibaba Group" (영어) → 같은 업체
    - "삼성전자" (한글) ↔ "Samsung Electronics" (영어) → 같은 업체
    - "株式会社ABC" (일본어) ↔ "ABC Corporation" (영어) → 같은 업체

### 3. 중복 판단 기준
- 공급업체명 유사도
- 주소 정보 유사도
- 국가 코드 일치
- **다국어 표기 동일 업체 판단**

### 4. 성능 요구사항
- 빠른 처리 (44k 행을 효율적으로 처리)
- 정확한 중복 탐지

### 5. 출력 요구사항
- 중복 그룹 탐지
- 중복 데이터 병합 (하나로 통합)
- 결과 파일 생성 (Excel)
- 상세 리포트

## 현재 방식의 문제점

1. ❌ **다국어 중복 미탐지**
   - Blocking 키가 언어별로 분리됨
   - "阿里巴巴"와 "Alibaba"가 다른 그룹으로 분리되어 비교되지 않음

2. ❌ **비효율적인 비교**
   - 큰 그룹에서 N² 비교 수행
   - 느린 라이브러리 사용 (fuzzywuzzy)

## 최적 해결 방법

### 추천: 하이브리드 방식

1. **다중 Blocking 키 생성**
   - 각 행에 대해 가능한 모든 언어 변형의 키 생성
   - 원본 키 + 로마자화 키 + 영어 변형 키 등
   - 같은 업체의 다른 언어 표기도 같은 그룹에 포함

2. **의미 기반 비교 (후보군에 대해서만)**
   - multilingual embedding 사용
   - 또는 로마자화 + 문자열 비교

3. **성능 최적화**
   - RapidFuzz 사용
   - Blocking으로 비교 횟수 대폭 감소
   - 2단계 접근법 (후보 추출 → 정교한 비교)

## 예상 성능

- **기존 방식**: 5-9시간 (다국어 중복 미탐지)
- **최적화 방식**: 30분 ~ 1시간 (다국어 중복 탐지 포함)
